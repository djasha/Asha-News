# Conflicts Dashboard (Public Product Note)

This page documents the public-facing behavior of `/conflicts`.

## Product Positioning

The Conflicts Dashboard is an analyst-style monitoring surface designed to prioritize verified data, source explainability, and explicit uncertainty for forecasts and theories.

## User-Facing Capabilities

1. Tactical comparison controls.
2. Verified-first KPI and trend boards.
3. Ranked related-news feed with signal explanations.
4. Analyst tab (hypotheses + probabilistic scenarios) when enabled.

## Public Safeguards

1. Identity redaction when verification/policy requirements are not met.
2. Non-deterministic framing for all forward-looking outputs.
3. Feature-flagged rollout controls.
4. Verified-first default means unreviewed events are excluded until editorial verification occurs.
